# Contributing to RaceSchedules

We welcome contributions to the RaceSchedules project! Your help is invaluable in making this application better for the entire motorsports community.

Please take a moment to review this document to understand how you can contribute.

## How to Contribute

There are several ways you can contribute to RaceSchedules:

1.  **Report Bugs:** If you find a bug, please open an issue on our GitHub repository. Provide a clear and detailed description of the bug, steps to reproduce it, and any relevant error messages.
2.  **Suggest Enhancements:** Have an idea for a new feature or an improvement to an existing one? Open an issue to discuss your suggestion.
3.  **Code Contributions:**
    *   Fork the repository.
    *   Create a new branch for your feature or bug fix: `git checkout -b feature/your-feature-name` or `git checkout -b bugfix/issue-description`.
    *   Make your changes, adhering to the existing code style and conventions.
    *   Write appropriate tests for your changes.
    *   Ensure all tests pass.
    *   Commit your changes with a clear and concise commit message.
    *   Push your branch to your fork.
    *   Open a Pull Request to the `main` branch of the original repository. Please provide a detailed description of your changes.
4.  **Documentation Improvements:** Help us improve our documentation, including `README.md` files, code comments, or any other relevant information.
5.  **Data Contributions:** Please refer to the [DATA_CONTRIBUTING.md](DATA_CONTRIBUTING.md) guide for contributing championship and schedule data.

## Code Style and Conventions

-   **Frontend (React/TypeScript):**
    *   Follow standard React best practices.
    *   Use TypeScript for type safety.
    *   Adhere to ESLint rules (run `npm run lint` in the `frontend` directory).
    *   Use Tailwind CSS for styling.
-   **Backend (FastAPI/Python):
    *   Follow PEP 8 guidelines for Python code.
    *   Use type hints for functions and variables.
    *   Adhere to any existing formatting (e.g., Black if configured) and linting rules.

## Setting Up Your Development Environment

Please refer to the `README.md` files in the `backend/` and `frontend/` directories for detailed instructions on setting up each part of the project.

Thank you for contributing to RaceSchedules!
